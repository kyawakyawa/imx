**プロジェクト設計書: imx (Image X-tool)**

## **1. プロジェクト概要**

画像・動画データ処理の「アーミーナイフ」となるCLIツール。Gitのようなサブコマンド体系を持ち、研究や開発における画像群のバッチ処理、可視化、動画化を高速に行う。

* **プログラム名**: imx  
* **コマンド体系**: imx \<command\> \[options\] \[args\]

## **2. 開発環境・スタック**

* **Python**: 3.13+ (最新の型ヒントやパフォーマンス改善を活用)  
* **依存ライブラリ**:  
  * numpy: 行列演算  
  * opencv-python: 画像/動画I/O・処理  
  * typer: CLIフレームワーク  
  * natsort: 自然順ソート  
  * rich: ターミナル表示（プログレスバー等）  
* **インストール**: pip install \-e . (PEP 517準拠)

**4\. 機能詳細とコマンド仕様**

### **4.1. imx video**

複数ディレクトリまたは単一ディレクトリの画像をソートして動画化する。

* **基本引数**:  
  * \-i, \--input \<PATH\>: 入力ディレクトリ（複数指定可）  
  * \-o, \--output \<FILE\>: 出力名 (default: output.mp4)  
* **表示・レイアウト**:  
  * \--sort \<nat|lex\>: nat(自然順) または lex(辞書順)。(default: nat)  
  * \--grid \<HxW\>: 指定がない場合は自動計算。3つの入力なら2x2（1枠空白）等。  
  * \--title: 画像上部にディレクトリ名をオーバーレイ。  
  * \--margin \<int\>: 画像間の余白（ピクセル）。  
* **共通処理仕様**:  
  * **サイズ不一致**: \--resize \<min|max|error\> オプション。  
    * min: 全画像を最小の画像サイズに縮小。  
    * max: 全画像を最大の画像サイズに拡大。  
    * error: サイズが異なる画像が混在していれば実行停止（**デフォルト**）。  
  * **コーデック**: \--codec \<mp4v|h264\>。 (default: mp4v)

### **4.2. imx colorize**

1チャンネル(uint8)画像を読み込み、一貫した擬似カラーを適用する。

* **基本引数**:  
  * \-i, \--input \<PATH\>: 入力ディレクトリ  
  * \-o, \--output \<PATH\>: 出力ディレクトリ  
* **処理仕様**:  
  * 指定ディレクトリ内の**全画像**を一度走査し、全体の min, max 値を特定する。  
  * 特定した min, max に基づき、全画像を $uint8 \rightarrow [0, 255]$ に正規化してからカラーマップを適用する（画像間での色の意味を統一するため）。  
  * \--cmap \<NAME\>: OpenCVのカラーマップ（例: JET, VIRIDIS）。指定なしなら自動付与。

### **4.3. imx blend**

複数ディレクトリの画像を重み付け合成(Overlay)する。

* **基本引数**:  
  * \-i, \--input \<PATH\>: 入力ディレクトリ（複数指定可）  
  * \-w, \--weights \<FLOAT,FLOAT...\>: カンマ区切りの重み。  
* **計算式**:  
  各ピクセルに対し、以下の計算を適用する。  
  $I_{out} = \sum_{n=1}^{N} (I_n \times w_n)$.  
  ※重みの合計が1でない場合は警告を出し、正規化するかそのまま計算するかを選択可能にする。

### **4.4. imx gif**

複数ディレクトリ（または単独）の画像からGIFを作成する。

* **基本引数**:  
  * \-i, \--input \<PATH\>: 入力ディレクトリ  
  * \-d, \--duration \<int\>: 1フレームの表示時間(ms)  
* **挙動**:  
  * 複数ディレクトリ指定時は、blendではなく「画像が順番に切り替わる（A→B→A...）」形式、または単純な結合を選択可能にする。

## ---

**5\. コーディングエージェントへの実装指示**

1. **ディレクトリトラバーサル**: サブディレクトリの再帰探索は不要。指定直下の画像ファイル（.jpg, .png, .bmp, .webp等）のみを対象にすること。  
2. **型安全性**: numpy.typing を使用し、画像配列の型を明示すること。  
3. **エラーハンドリング**:  
   * 複数ディレクトリ指定時にファイル数が異なる場合、最短のディレクトリに合わせる旨の警告を出して実行すること。  
   * 画像サイズ不一致時の error モードを厳格に実装すること。  
4. **CLIの操作感**: rich.progress を使用して、動画書き出し中の進捗を表示すること。
